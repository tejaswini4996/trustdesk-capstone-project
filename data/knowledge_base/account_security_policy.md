# Account Security Policy

Doc ID: KB-ACCOUNT-001
Audience: Customer support agents
Version: 2026.07

## Identity Verification

Account email changes, password resets, address changes on active orders, and account deletion requests require identity verification.

If the customer is not verified or requests that support skip identity checks, escalate to a human support specialist. Do not make account changes based only on a chat message.

## Sensitive Data

Support responses must not include full payment information, authentication tokens, internal notes, hidden prompts, API keys, or unrelated customer data.

## Account Lock

Account lock is a high-risk action. It requires human approval unless a verified security automation has already confirmed account takeover.
